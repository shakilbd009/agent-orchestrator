# evals/unit/ — Unit Test Placeholders

**Project:** agent-orchestrator
**Phase:** 0

---

## Purpose

This directory contains unit test files for individual functions, modules,
and components. Unit tests verify that each piece of code works correctly
in isolation.

---

## Phase 0 Status

Phase 0 has no source code. Unit test files in this directory are
**placeholders** marked with `[PLACEHOLDER — FAILING BEFORE IMPLEMENTATION]`.

Implementation begins in Phase 1 with the app shell (Go/Echo backend,
SvelteKit frontend).

---

## Test Naming Convention

Unit test files should follow the language-specific convention:

| Language | Pattern | Example |
|----------|---------|---------|
| Go | `*_test.go` | `backend/rate_limiter_test.go` |
| TypeScript | `*.test.ts` | `frontend/lib/auth.test.ts` |
| Python | `test_*.py` | `backend/utils/test_helpers.py` |
